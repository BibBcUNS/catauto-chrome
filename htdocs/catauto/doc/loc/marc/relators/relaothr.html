Introduction

This part contains various lists of codes assigned to be used to identify sources of data in MARC fields or to characterize the data in the MARC field. The codes are recorded in subfield $2 of the MARC variable field to which they apply. These code lists grouped here do not fit into any of the existing groupings of codes in Parts I-VI.

Arrangement of the List

The lists of codes are grouped according to the numeric MARC 21 field in which they can be used. Entries in each list are arranged in alphabetical order by the code and consist of the code followed by a citation for the source, or information that defines the code.

Code Structure

Codes assigned to sources are usually derived from a title. Each code consists of a maximum of eight lowercase characters. When the code is not for a source (e.g., access method), it is based on the descriptive name or caption associated with it (e.g., "http" is based on the name of the Hypertext Transfer Protocol).

The code may be followed by the edition, language, or date of publication of the source work. The code is separated from the addition by a slash (/). Instead of the full name of a language, the 3-character MARC language code should be used when qualification by language is needed. Omission of the edition, language, or year of publication implies that the assigning institution used the most recent edition of the sourcein its original language (usually the language of the title itself). Example:

hlasja/1997
HLAS journal abbreviations, in Handbook of Latin American studies, 1997 edition.

FIELD 042 (Authentication Code)

dc - Dublin Core
Code dc signifies that the record was converted from a simple resource description record in another syntax (e.g., HTML or SGML) using the Dublin Core metadata element set. The content of the record (descriptive elements and headings) may or may not follow any cataloging standard (e.g., AACR2, ISBD).
dhca - Dance Heritage Coalition Access Project
Code dhca signifies that the record was authenticated under the auspices of the Dance Heritage Coalition Access Project.
dlr - Digital library registry
Code dlr signifies that the record was authenticated under the auspices of a digital library registry.
gamma - Georgia Archives & Manuscripts Automated Access Project
Code gamma signifies that the record was authenticated under the auspices of the Georgia Archives & Manuscripts Automated Access Project, coordinated by the University Center in Georgia.
gils - Government Information Location Service
Code gils signifies that the record has been derived from a locator record contained in a GILS server using the GILS Application Profile. The content of the record (descriptive elements and headings) may or may not follow any cataloging standards (e.g., AACR2, ISBD).
isds/c - ISSN Canada
Code isds/c signifies that the CONSER ISSN Network data elements have been verified by ISSN Canada. ISSN Canada authenticates ISSN data elements for Canadian imprints. The code reflects the ISSN Network's earlier name, ISDS (International Serials Data System).
issnuk - ISSN UK Centre
Code issnuk signifies that the ISSN Network data elements have been verified by ISSN UK. ISSN UK authenticates ISSN data elements for United Kingdom imprints.
lc -Library of Congress
Code lc signifies that the CONSER descriptive elements and all headings have been verified by the Library of Congress. LC authenticates records not within the purview of the National Library of Canada.
lcac - LC Annotated Children's Cataloging Program
Code lcac signifies that the cataloging record was handled as part of the Library of Congress Annotated Children's Cataloging Program whereby summaries and children's subject headings are added to facilitate the use of the item by a juvenile audience.
lccopycat - LC Copy Cataloging
Code lccopycat signifies that the Library of Congress has used another organization's record as the basis for its cataloging. All headings have been checked against the relevant authority file (except subject headings not from Library of Congress Subject Headings).
lccopycat-nm - LC Copy Cataloging-Near Match
Code lccopycat-nm signifies that the Library of Congress has used an existing bibliographic record that is not an exact match of the item in hand as the basis for its cataloging. The record copied could be from LC or another cataloging agency. The record copied could be for another edition, version, or format of the item. The data are updated to fully reflect the item in hand. All headings are checked against the relevant authority file (except subject headings not from Library of Congress Subject Headings).
lcd -CONSER full authority application
Code lcd signifies that the CONSER descriptive elements and all headings have been verified by the Library of Congress or an authenticating CONSER participant.
lcderive - LC derived cataloging
Code lcderive signifies that the Library of Congress has used another organization's catalog record essentially "as is" for its cataloging. All headings and uniform titles have been processed through special software developed to evaluate and ensure that the headings and uniform titles match all existing LC authority records and conform to cataloging rules.
lchlas - LC Handbook of Latin American Studies
Code lchlas signifies that the descriptive and subject elements have been derived from the internal Library of Congress Handbook of Latin American Studies file. Name headings may or may not have been verified against LC's name authority file.
lcllh - LC Law Library Hispanic
Code lcllh signifies that the descriptive elements have been derived from the internal Library of Congress Index to Hispanic Legislation. Name headings have been verified against LC's name authority file.
lcnccp - LC National Coordinated Cataloging Program
Code lcnccp signifies that the record is a Library of Congress National Coordinated Cataloging Program record. The headings are all present in LC's authority files and the descriptive elements follow AACR 2.
lcnitrate - LC Nitrate Film
Code lcnitrate signifies that the descriptive elements have been derived from a Library of Congress database containing inventory information on LC's nitrate film collection. Name headings have not been verified against LC's name authority file.
lcnuc - National Union Catalog
Code lcnuc signifies that descriptive elements have been edited and all headings verified against the relevant authority file to prepare the records for inclusion in an LC National Union Catalog file.
lcode - LC Overseas Data Entry
Code lcode signifies that the record is from one of LC's overseas field offices. The descriptive elements follow AACR 2 and the headings have been checked off-line against LC's name authority file using MARC-Names (on CD-ROM).
msc - CONSER minimal authority application
Code msc signifies that the CONSER descriptive elements have been verified by the Library of Congress or an authenticating CONSER participant. All headings have been checked against the relevant authority file, but an authority record was not found for one or more headings.
nlc - National Library of Canada
Code nlc signifies that the CONSER descriptive elements and headings have been verified by the National Library of Canada. NLC authenticates records for Canadian imprints and records of Canadian interest.
nlmcopyc
- National Library of Medicine Copy Cataloging
Code nlmcopyc signifies that National Library of Medicine has used another organization's record as the basis for its cataloging. All headings have been checked against the relevant authority file (except subject headings not from the National Library of Medicine).
nsdp - National Serials Data Program
Code nsdp signifies that the ISSN Network data elements have been verified by the United States National Serials Data Program. NSDP authenticates ISSN Network data elements for U.S. imprints. NSDP also authenticates ISSN assignments and key titles for records received from other ISSN Network centers, with the exception of Canadian imprints authenticated by ISSN Canada.
ntccf - LC National Translations Center Citation File
Code ntccf signifies that the descriptive elements have been derived from data in the Library of Congress National Translations Center Citation File. Name headings have not been verified against LC's name authority file.
pcc - Program for Cooperative Cataloging
Code pcc signifies that the record was authenticated under the auspices of the Program for Cooperative Cataloging. All name and series headings have been checked against the appropriate national level authority file and authority records have been created if they did not already exist. Subject headings are checked for authorized forms or combinations supported by the relevant authority.
premarc - LC PreMARC Retrospective Conversion Project
Code premarc signifies that the record was converted to machine-readable form as part of a Library of Congress retrospective conversion project. The source records used in the initial phase of this project (1978-1979) were printed catalog cards taken from the LC general shelflist, atlas and map shelflists, and the music shelflist. Subsequently, printed catalog cards from various other manual LC files were used as the source for additional records converted.
reveal - REVEAL Union Catalog Project
Code reveal signifies that the MARC record conforms to the criteria for inclusion in the REVEAL union catalog.
sanb - South African National Bibliography Project
Code sanb signifies that the descriptive elements and all headings in the records have been authenticated for inclusion in the South African National Bibliography Project.
sanb - South African National Bibliography Project
Code sanb signifies that the descriptive elements and all headings in the records have been authenticated for inclusion in the South African National Bibliography Project.
ukblsr - British Library Standard Record
Code ukblsr signifies that the record conforms to the definition for the British Library Standard Record and all name and subject headings have been verified against the relevant authority file. [use only after June 16, 2004]
ukscp - UK Legal Deposit Libraries' Shared Cataloguing Programme Record
Code ukscp indicates that the record conforms to the definition for the UK Legal Deposit Libraries' Shared Cataloguing Programme Record. [use only after June 16, 2004]
xissnuk - Unverified by ISSN UK Centre
Code issnuk signifies that the ISSN Network data elements have not been verified by ISSN UK. ISSN UK authenticates ISSN data elements for United Kingdom imprints.
xlc - LC does not consider item a serial
xnlc - NLC does not consider item a serial

xnsdp - NSDP does not consider item a serial
A code beginning with the letter "x" signifies that the particular agency involved did not consider the CONSER item to be a serial at the time the publication was evaluated for authentication. However, if subsequent developments or information warrant it, the title may be recataloged as a serial and the record authenticated.
